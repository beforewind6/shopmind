### 2.3 在线推理：一条请求的完整旅程

以 `"蓝牙耳机的续航时间？"` 为例：

```

│  POST /api/v1/chat {"message": "蓝牙耳机的续航时间？"}                │

│  ┌─ 0ms ─┐  Flask 接收 JSON → 提取 message                          │
│  │        │                                                          │

│  │        │                                                          │
│  │        │  ┌─────────────────────────────────────────────┐         │
│  │        │  │  Step 1: MockLLM 推理                        │         │
│  │        │  │                                              │         │
│  │        │  │  检测到关键词 "蓝牙" → 路由到商品知识问答     │         │
│  │        │  │                                              │         │
│  │        │  │  Thought: "用户想了解商品规格和功能参数"      │         │
│  │        │  │  Action:  商品知识问答                       │         │
│  │        │  │  Action Input: "蓝牙耳机的续航时间？"        │         │
│  │        │  └──────────────────────┬──────────────────────┘         │
│  │        │                         │                                │
│  │        │  ┌──────────────────────▼──────────────────────┐         │
```

## 四、检索融合算法详解

```
Query: "蓝牙耳机的续航时间？"
         │
         ├──→ Dense (稠密检索)
         │    FallbackEmbedder → 384维向量
         │    Cosine 相似度 → Top-20
         │    [
         │      {id: "product_manual_3", score: 0.87, text: "蓝牙版本: 5.3..."},
         │      {id: "faq_7",           score: 0.62, text: "Q: 蓝牙耳机..."},
         │      ...
         │    ]
         │
         ├──→ Sparse (稀疏检索)
         │    jieba分词: ["蓝牙", "耳机", "的", "续航", "时间"]
         │    BM25 评分 → Top-20
         │    [
         │      {id: "bm25_12", score: 9.2, text: "蓝牙版本: 5.3..."},
         │      {id: "bm25_5",  score: 6.8, text: "续航: 6-8小时..."},
         │      ...
         │    ]
         │
         └──→ RRF 融合 (k=60)
              ┌─────────────────────────────────────────────┐
              │  score[doc] = 1/(60+rank_dense)            │
              │             + 1/(60+rank_sparse)            │
              │                                             │
              │  例如 product_manual_3:                      │
              │    rank_dense=1 → 1/61 = 0.0164             │
              │    rank_sparse=1 → 1/61 = 0.0164            │
              │    total = 0.0328 (最高分)                   │
              │                                             │
              │  按融合分降序 → Top-3                        │
              └─────────────────┬───────────────────────────┘
                                │
                    返回 3 个最佳匹配文档
                    ① 产品手册-蓝牙耳机
                    ② 产品手册-续航详情
                    ③ FAQ-物流支付 (被后续过滤掉)
```

## 十、与 ReviewShield 的差异对比

| 维度 | ReviewShield | ShopMind |
|------|-------------|----------|
| 核心架构 | BERT分类 + RAG 审核 | ReAct Agent + RAG 问答 |
| AI 角色 | 审核员（判断违规则） | 客服（回答用户提问） |
| 交互模式 | 单向审核 | 双向对话 |
| 工具数量 | 无（单链 Pipeline） | 双工具协同 |
| LLM 调用 | 1 次/条 | 2 次/轮（Thought + 综合） |
| Web 框架 | FastAPI | Flask |
| 前端 | 自定义 HTML | Streamlit |
| 评估 | RAGAS 四维 | 三维 (检索/生成/性能) |
| 向量检索 | Hybrid + RRF + Rerank | Hybrid + RRF |
| 数据规模 | 200 条评论 | 24 个知识块 |
